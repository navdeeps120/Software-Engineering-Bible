# Project Goal

This repository aims to become the most comprehensive Software Engineering learning repository possible.

It is **NOT** intended to be another notes repository.

It is intended to become a complete software engineering curriculum that teaches concepts from first principles through understanding, implementation, optimization, documentation and production projects.

The repository should eventually be suitable for public release on GitHub.

Every chapter should be production-quality.

---

# Philosophy

We never learn technologies.

We learn why technologies exist.

Every topic should answer

- What is it?
- Why was it invented?
- What problem does it solve?
- How does it work internally?
- What are the tradeoffs?
- When should I use it?
- When should I not use it?
- How would I implement one myself?
- How would large companies use it?

Professional.

Beginner friendly.

Never oversimplify.

Avoid unnecessary jargon.
